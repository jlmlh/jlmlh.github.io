现代JavaScript教程学习笔记

前言

　　本学习笔记主要是记录在学习《现代JavaScript教程》时的一些知识点、习题和想法，主要以便于理解和记忆的提纲为主，同时对一些语法和代码做一摘录，但是不大篇幅地抄录书上的内容。

第一部分 JavaScript编程语言

第一章 简介

一、JavaScript 简介

  1. 一种可以被直接写在网页的 HTML 中的编程语言，写出来的程序被称为脚本；
  2. 目的是为了使网页更生动；
  3. 脚本被以纯文本的形式提供和执行。它们不需要特殊的准备或编译即可运行；
  4. JavaScript 和 Java 之间没有任何关系；
  5. JavaScript 不仅可以在浏览器中执行，也可以在服务端执行，甚至可以在任意搭载了 JavaScript 引擎 的设备；
  6. 浏览器中嵌入了 JavaScript 引擎，有时也称作“JavaScript 虚拟机”，不同的引擎有不同的“代号”，例如：
    • V8 —— Chrome 和 Opera 中的 JavaScript 引擎。
    • SpiderMonkey —— Firefox 中的 JavaScript 引擎。
    • ……还有其他一些代号，像 “Chakra” 用于 IE，“ChakraCore” 用于 Microsoft Edge，“Nitro” 和 “SquirrelFish” 用于 Safari，等；
  7. 如今，JavaScript 已经成为了与 HTML/CSS 完全集成的，使用最广泛的浏览器语言。
  8. 有很多其他的语言可以被“编译”成 JavaScript，这些语言还提供了更多的功能。

二、手册与规范

  1. ECMA-262 规范 ： https://tc39.es/ecma262/；
  2. MDN（Mozilla）JavaScript 索引： https://developer.mozilla.org/en-US/docs/Web/JavaScript/Reference ；
  3. MSDN ：http://msdn.microsoft.com/；
  4. http://caniuse.com —— 每个功能的支持表；
  5. https://kangax.github.io/compat-table —— 一份列有语言功能以及引擎是否支持这些功能的表格；

第二章 JavaScript 基础知识

一、Hello, world!

  1. JavaScript 程序可以在 <script> 标签的帮助下插入到 HTML 文档的任何地方；

  2. 大量的 JavaScript 代码可以放入一个单独的文件，脚本文件可以通过 src 特性（attribute）添加到 HTML 文件中；

  3. 只有最简单的脚本才嵌入到 HTML 中，更复杂的脚本存放在单独的文件中；

  4. 脚本只会下载一次，后续会从缓冲区读取；

  5. 一个单独的 <script> 标签不能同时有 src 特性和内部包裹的代码，可以将src放在第二个 <script> 标签中；

三、现代模式，“use strict”

  1. 字符串 "use strict" 或 'use strict' 处于脚本文件的顶部时，整个脚本文件都将以“现代”模式进行工作；

  2. "use strict" 可以被放在函数体的开头，这样则可以只在该函数中启用严格模式；

  3. "use strict" 不在脚本的最顶部时，严格模式可能无法启用；

  4. 使用后，没有办法取消 use strict；

四、变量

  1. 在 JavaScript 中用 let 关键字创建变量；

  2. 用赋值运算符 = 为变量添加数据；

  3. var 关键字与 let 大体相同，也用来声明变量，但稍微有些不同；

  4. 一个变量应该只被声明一次；

  5. 变量名称必须仅包含字母，数字，符号 $ 和 _，且首字符必须非数字；

  6. 变量名称区分大小写；

  7. 变量名称不能使用保留字；

  8. 可以不使用 let 进行变量声明，但是在严格模式下会报错；

  9. 声明一个常量用 const 而不是 let ；

  10. 大写命名的常量仅用作“硬编码（hard-coded）”值的别名；

  11. 一些变量命名常见的规则：

    • 使用易读的命名，比如 userName 或者 shoppingCart；
    • 离诸如 a、b、c 这种缩写和短名称远一点，除非你真的知道你在干什么；
    • 变量名在能够准确描述变量的同时要足够简洁。不好的例子就是 data 和 value，这样的名称等于什么都没说。如果能够非常明显地从上下文知道数据和值所表达的含义，这样使用它们也是可以的；
    • 脑海中的术语要和团队保持一致。如果网站的访客称为“用户”，则我们采用相关的变量命名，比如 currentUser 或者 newUser，而不要使用 currentVisitor 或者一个 newManInTown。

  12. 新建变量比重用变量利大于弊；

五、数据类型

1、Number类型

  1. number 类型代表整数和浮点数；
  2. number 包括“特殊数值：Infinity、-Infinity 和 NaN；
  3. Infinity 代表数学概念中的无穷大；
  4. NaN 代表一个计算错误，具有粘性；
  5. number”类型无法表示大于 (2^53-1)（即 9007199254740991），或小于 -(2^53-1) 的整数。

2、BigInt 类型

  1. BigInt 类型表示任意长度的整数；

  2. 可以通过将 n 附加到整数字段的末尾来创建 BigInt 值；

    3、String类型

  3. JavaScript 中的字符串必须被括在引号里；

  4. 有三种包含字符串的方式：

    • 双引号："Hello"
    • 单引号：'Hello'
    • 反引号：Hello

  5. 反引号是 功能扩展 引号，它允许我们通过将变量和表达式包装在 ${…} 中，来将它们嵌入到字符串中；

  6. ${…} 内的表达式会被计算，计算结果会成为字符串的一部分；

  7. JavaScript 中没有 character 类型；

4、Boolean类型

  1. boolean 类型仅包含两个值：true 和 false；

5、null 值

  1. JavaScript 中的 null 仅仅是一个代表“无”、“空”或“值未知”的特殊值；

  2. JavaScript 中的 null 不是一个“对不存在的 object 的引用”或者 “null 指针”；

    6、undefind 值

  3. 如果一个变量已被声明，但未被赋值，那么它的值就是 undefined；

    7、object 类型

  4. object 类型用于储存数据集合和更复杂的实体；

8、 symbol 类型

  1. symbol 类型用于创建对象的唯一标识符；

9、typeof 运算符

  1. typeof 运算符返回参数的类型；

  2. 它支持两种语法形式：

    • 作为运算符：typeof x；
    • 函数形式：typeof(x)；

  3. 对 typeof x 的调用会以字符串的形式返回数据类型：

  4. Math 是一个提供数学运算的内建 object；

  5. typeof null 的结果是 "object"。这是官方承认的 typeof 的行为上的错误，这个问题来自于 JavaScript 语言的早期，并为了兼容性而保留了下来。null 绝对不是一个 object。null 有自己的类型，它是一个特殊值。

七、类型转换

  1. 运算符和函数会自动将赋予它们的值转换为正确的类型；

  2. alert(value) 将 value 转换为字符串类型；

  3. String(value) 显式地将 value 转换为字符串类型；

  4. 在算术函数和表达式中，会自动进行 number 类型转换；

  5. Number(value) 显式地将 value 转换为 number 类型；

  6. string类型转为数字时需要进行显式转换，但如果该字符串不是一个有效的数字，转换的结果会是 NaN；

    number 类型转换规则：

    值 转换后
    undefined NaN
    null 0
    true 和 false 1 and 0
    string 去掉首尾空格后的纯数字字符串中含有的数字。如果剩余字符串为空，则转换结果为 0。否则，将会从剩余字符串中“读取”数字。当类型转换出现 error 时返回 NaN。

  7. boolean类型转换中，空值（如 0、空字符串、null、undefined 和 NaN）将变为 false，其他值变成 true；

  8. 任何非空字符串都是 true 包含0的字符串 "0" 是 true；

    八、基础运算符

  9. 运算元是运算符应用的对象；

  10. 如果一个运算符对应的只有一个运算元，那么它是一元运算符；

  11. 如果一个运算符拥有两个运算元，那么它是二元运算符；

  12. 常见的数学运算有加法 + ，减法 - ，乘法 * ，除法 / ，取余 % 和求幂 ** ；

  13. 求幂运算 a ** b 是 a 乘以自身 b 次；

  14. 用二元运算符 + 连接字符串；

  15. 二元 + 是唯一一个以这种方式支持字符串的运算符，其他算术运算符只对数字起作用，并且总是将其运算元转换为数字。

  16. 一元运算符加号 + 应用于单个值，对数字没有任何作用，但是如果运算元不是数字，加号 + 则会将其转化为数字；

  17. 一元运算符 + 效果和 Number(...) 相同，但是更加简短；

  18. 修改并赋值运算符 += ， *= ，/= 和 -= 优先级比较低；

  19. 自增/自减运算符 ++ 和 -- 只能应用于变量；

  20. 如果我们想要对变量进行自增操作，并且需要立刻使用自增后的值，那么我们需要使用前置形式；

  21. 如果我们想要将一个数加一，但是我们想使用其自增之前的值，那么我们需要使用后置形式；

  22. 位运算符有：

    • 按位与 ( & )
    • 按位或 ( | )
    • 按位异或 ( ^ )
    • 按位非 ( ~ )
    • 左移 ( << )
    • 右移 ( >> )
    • 无符号右移 ( >>> )

  23. 逗号运算符 , 优先级很低；

九、值的比较

  1. 比较运算符始终返回布尔值；
  2. 字符串的比较，会按照“词典”顺序逐字符地比较大小；
  3. 当对不同类型的值进行比较时，它们会先被转化为数字（不包括严格相等检查）再进行比较；

  5. 在使用 > 或 < 进行比较时，需要注意变量可能为 null/undefined 的情况。比较好的方法是单独检查变量是否等于null/undefined；
  6. undefined 不应该被与其他值进行比较

    • 如果 a 不是已定义的，则结果为 b。

  3. 可以使用 ?? 序列从一系列的值中选择出第一个非 null/undefined 的值；

  4. ?? 在 = 和 ? 之前计算，但在大多数其他运算符（例如，+ 和 *）之后计算；

  5. JavaScript 禁止将 ?? 运算符与 && 和 || 运算符一起使用，除非使用括号明确指定了优先级；

第四章、Object（对象）：基础知识

一、对象

  1. 对象则用来存储键值对和更复杂的实体；

  2. 可以使用点符号访问属性值，属性的值可以是任意类型；

  3. 可以用 delete 操作符移除属性；

  4. 可以用多字词语来作为属性名，但必须加上引号；

  5. 列表中的最后一个属性应以逗号结尾，叫做尾随（trailing）或悬挂（hanging）逗号；

  6. 用 const 声明的对象能被修改；

  7. const 声明仅固定了 user 的值，而不是值（该对象）里面的内容；

  8. 访问多字词语的属性时，需要用方括号 [] ；

  9. 多字词语的属性可以是根据用户的输入得到的，但点符号不可以；

    点符号不能以类似的方式使用：

  10. 当创建一个对象时，如果在对象字面量中使用方括号，该属性叫做计算属性，含义是属性名应该从变量中获取；

  11. 属性命名没有限制，可以是任何字符串或者 symbol，其他类型会被自动地转换为字符串；

  12. 读取不存在的属性只会得到 undefined；

  13. 操作符 "in" 检查属性是否存在的；

  14. “for…in” 循环：遍历一个对象的所有键（key）；

  15. 整数属性会被进行排序，其他属性则按照创建的顺序显示；

二、对象引用和复制

  1. 对象是通过引用被存储和复制的，这是与原始类型的根本区别；

  2. 赋值了对象的变量存储的不是对象本身，而是该对象在内存中的地址；

  3. 当一个对象变量被复制时，肤质的只是引用，对象本身并没有被复制；

  4. 仅当两个对象为同一对象时，两者才相等；

  5. 两个独立的对象不相等；

  6. 可以使用 Object.assign 方法来真正复制对象：

    • 参数 dest 是指目标对象；
    • 参数 src1, ..., srcN（可按需传递多个参数）是源对象；
    • 从第二个开始的所有参数的属性都被拷贝到第一个参数的对象中；
    • 调用结果返回 dest；

  7. 如果被拷贝的属性的属性名已经存在，那么它会被覆盖；

  8. 深层克隆中， Object.assign(dest, [src1, src2, src3...]) 返回的只是对象地址，并不是克隆对象，深拷贝可以用 _.cloneDeep(obj) 函数；

    三、垃圾回收

  9. 可达值是以某种方式可访问或可用的值它们一定是存储在内存中的；

  10. 如果一个值可以通过引用或引用链从根访问任何其他值，则认为该值是可达的；

  11. 一些固有的可达值的基本集合：

    • 当前函数的局部变量和参数；
    • 嵌套调用时，当前调用链上所有函数的变量与参数；
    • 全局变量；
    • ······

    这些值被称作根（roots），明显不能被释放；

  12. 在 JavaScript 引擎中有一个被称作垃圾回收器的东西在后台执行，它监控着所有对象的状态，并删除掉那些已经不可达的；

  13. 几个对象相互引用，但外部没有对其任意对象的引用，这些对象也可能是不可达的，并被从内存中删除；

  14. 被引用与可访问（从一个根）不同：一组相互连接的对象可能整体都不可达；
